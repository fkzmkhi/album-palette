既存の album-palette アプリ ( https://github.com/fkzmkhi/album-palette ) に
"Today's Music Color" 機能を追加してほしい。
ジャンルを選んで、そのジャンルの人気アルバム TOP10 をジャケット + パレットで表示する機能。

## 仕様

### UI 構成 (page.tsx 再構成)

画面上から下に:

1. 検索バー (既存)
2. ジャンルセレクター (新規) - チップ群、横スクロール可
3. **Today's Music Color - {Genre}** セクション (新規) - 5×2 = 10件のカードグリッド
4. 検索フォーカス時のみ: 検索結果 (既存) を3の上 or オーバーレイで表示
5. アルバム選択時の詳細パレット (既存)
6. 履歴 (既存)

### ジャンルセレクター

- 固定リスト:
  ```ts
  export const GENRES = [
  	"pop",
  	"rock",
  	"j-pop",
  	"hip-hop",
  	"electronic",
  	"jazz",
  	"indie",
  	"metal",
  	"classical",
  	"r&b",
  	"ambient",
  ] as const;
  ```
- デフォルト選択: "pop"
- チップ UI、選択中は強調表示
- チップクリックで TOP10 を再取得

### Today's Music Color セクション

- セクションタイトル: `Today's Music Color - {Genre}` (英語固定)
- カード10件をグリッド (sm:2列 md:3列 lg:5列)
- 各カード:
  - ジャケ画像（大、aspect-square）
  - 下に5色パレット（横並びの細いバー、各色クリックでHEXコピー）
  - アルバム名・アーティスト名
  - クリックで既存の AlbumDetail に遷移（パレットは取得済みのものを再利用）
- ロード中はスケルトン表示（10件分）

### API: /api/genre-top

- Route Handler: `app/api/genre-top/route.ts`
- クエリ: `?tag={genre}&limit=10`
- Last.fm の `tag.getTopAlbums?tag={tag}&limit=10` を呼ぶ
- API キーは .env.local の LASTFM_API_KEY
- レスポンスを次の形に整形して返す:
  ```ts
  type GenreTopResponse = {
  	genre: string;
  	fetchedAt: string; // ISO datetime
  	items: Array<{
  		rank: number;
  		name: string;
  		artist: string;
  		mbid?: string;
  		imageUrl: string;
  	}>;
  };
  ```
- Route Handler に `export const revalidate = 3600;` で1時間キャッシュ

### パレット抽出

- カードごとに colorthief で5色抽出 (既存ロジック流用)
- 画像 onLoad 後に順次抽出、抽出済みのものから順に表示
- `crossOrigin="anonymous"` 必須

### "Today's" の扱い

- ラベルだけ "Today's"、実態は Last.fm の最新人気
- README に「Last.fm の最新人気から取得」と注記
